# WebSSH 项目操作手册

欢迎使用 WebSSH，这是一个基于 Spring Boot 的企业级 Web 终端工具。本手册将引导您从基础入门到掌握高级功能。

---

## 1. 快速开始

### 1.1 登录系统
- **访问地址**: 默认部署在 `http://your-server-ip:8080`
- **默认账户**: `admin`
- **默认密码**: `admin123`
> **安全建议**: 首次部署后，请务必在 `application.properties` 或环境变量中修改默认凭据。

### 1.2 创建首个 SSH 会话
1. 点击左侧侧边栏的 **“添加”** 按钮。
2. 在右侧弹出的“会话配置”面板中，填入：
   - **会话名**: 便于识别的名称（如：开发机-A）。
   - **主机地址**: 服务器 IP 或域名。
   - **端口**: 默认 22。
   - **认证方式**: 选择“密码”或“私钥”。
3. 点击底部的 **“保存”**（若需后续快速连接）或直接点击 **“连接会话”**。

---

## 2. SSH 终端操作

### 2.1 多标签并行
- **新建标签**: 点击顶部工具栏的 `+` 图标，可同时连接多个不同的服务器，或同一服务器的新连接。
- **切换标签**: 直接点击顶部标签页。

### 2.2 界面与主题
- **配色切换**: 侧边栏顶部提供“终端配色”下拉框，支持经典蓝、复古琥珀、暗夜紫等 6 种主题。
- **全屏模式**: 工具栏点击“全屏”按钮，沉浸式命令行体验。
- **自适应**: 浏览器窗口大小变化时，终端尺寸会自动同步至后端。

### 2.3 右键菜单
- 终端区域支持右键点击进行 **复制** 和 **粘贴** 操作（部分浏览器需要授权）。

---

## 3. SFTP 文件管理

### 3.1 开启面板
- 点击工具栏右侧的 **“SFTP”** 按钮打开文件管理面板。

### 3.2 基础操作
- **浏览**: 双击文件夹进入，点击“上级”返回。
- **下载**: 点击文件右侧的下载图标。支持超大文件分片确认传输，稳定可靠。
- **上传**: 点击“上传文件”按钮，支持大文件切片上传。
- **同步**: 终端执行 `cd` 命令切换目录后，SFTP 路径会自动刷新同步至最新 `$PWD`。

---

## 4. SSH 端口转发（隧道）

1. 在会话配置的“端口转发”卡片中：
   - **方向**: 选择“本地转发（L）”或“远程转发（R）”。
   - **绑定地址/端口**: 本地监听的 IP 和端口。
   - **目标地址/端口**: 隧道另一端的目标。
2. 连接 SSH 会话后，转发规则将自动生效。
3. 您可以在“端口转发”列表中实时添加或删除规则。

## 5. 机器人与 AI 集成

### 5.1 机器人配置
点击侧边栏左下角的 **“机器人设置”**：

- **Telegram**: 填入 `@BotFather` 提供的 Token，启用后可通过 Telegram 远程管理。
- **QQ 官方机器人**: 填入 AppID 和 Secret，启用后支持私聊指令。
- **微信 ClawBot**: 
  1. 点击 **“扫码获取 Token”**。
  2. 手机微信扫码并确认授权。
  3. 系统自动回填 Token，勾选“启用”并保存。

### 5.2 常用机器人指令
在机器人对话框中输入：
- `/list`: 列出已保存的 SSH 会话及其状态。
- `/connect [ID]`: 连接指定会话并接收输出。
- `/disconnect`: 主动断开当前连接。
- `/codex [Prompt]`: 进入 Codex AI 编程模式，之后发送的所有文本将由 AI 处理并执行。
- `/claude [Prompt]`: 进入 Claude AI 模式。
- `codex_stop`: 退出 AI 模式。

## 6. 系统配置与安全

### 6.1 凭据加密
- 保存会话时勾选 **“保存加密凭据”**，您的 SSH 密码和私钥将通过 **AES-GCM** 算法加密后存储在本地 JSON 文件中（`data/sessions/`），不会以明文暴露。

### 6.2 主机指纹 (Host Fingerprint)
- 首次连接未知主机时，系统会自动记录其指纹。后续连接若指纹发生变化，系统将拒绝连接以防中间人攻击。

### 6.3 资源限制 (Governance)
- 系统内置资源治理（可在 `application.properties` 调整）：
  - 每个用户最大并发会话数。
  - AI 任务执行超时限制。
  - 机器人消息聚合回复（防止触发平台频控）。

## 7. 常见故障处理

- **无法连接旧服务器**: 连接报错 `Algorithm negotiation fail (ssh-rsa)`？
  - *解决方法*: 在 `application.properties` 中设置 `webssh.ssh.allow-legacy-ssh-rsa=true`。
- **端口被占用**: 提示 `Port already in use`？
  - *解决方法*: 修改 `server.port` 配置。
- **会话丢失**:
  - *原因*: 默认数据存储在项目根目录 `data/` 文件夹下，请确保容器部署时对该目录做了持久化挂载。
